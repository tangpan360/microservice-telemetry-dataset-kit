# Traffic Pipelines

This document describes how to download, decompress, and extract second-level request count time series from traffic datasets.

## NASA-HTTP

### Download

Official source:

- [NASA-HTTP - Internet Traffic Archive](https://ita.ee.lbl.gov/html/contrib/NASA-HTTP.html)

Download:

```bash
mkdir -p dataset/raw/NASA-HTTP
wget -O dataset/raw/NASA-HTTP/NASA_access_log_Aug95.gz \
  ftp://ita.ee.lbl.gov/traces/NASA_access_log_Aug95.gz
```

Decompress:

```bash
gunzip dataset/raw/NASA-HTTP/NASA_access_log_Aug95.gz
```

### Extract

By default, we extract **weekday-only** second-level data from `NASA_access_log_Aug95` for `1995-08-07` to `1995-08-20`. Weekends are skipped, resulting in `10` days:

```bash
python pipelines/traffic/extract_nasa_requests.py
```

Default output:

- `dataset/processed/traffic/NASA-HTTP/NASA_access_log_Aug95_19950807_19950820_weekdays_requests_per_second.csv`

To include weekends:

```bash
python pipelines/traffic/extract_nasa_requests.py --include-weekends true
```

### Plot

Pass the dataset name:

```bash
python pipelines/traffic/plot_daily_minute_curves.py --data nasa
```

This reads from:

- `dataset/processed/traffic/NASA-HTTP`

and writes plots to the same directory:

- `dataset/processed/traffic/NASA-HTTP`

## ClarkNet-HTTP

### Download

Official source:

- [ClarkNet-HTTP - Internet Traffic Archive](https://ita.ee.lbl.gov/html/contrib/ClarkNet-HTTP.html)

Download:

```bash
mkdir -p dataset/raw/ClarkNet-HTTP
wget -O dataset/raw/ClarkNet-HTTP/clarknet_access_log_Aug28.gz \
  ftp://ita.ee.lbl.gov/traces/clarknet_access_log_Aug28.gz
wget -O dataset/raw/ClarkNet-HTTP/clarknet_access_log_Sep4.gz \
  ftp://ita.ee.lbl.gov/traces/clarknet_access_log_Sep4.gz
```

Decompress:

```bash
gunzip dataset/raw/ClarkNet-HTTP/clarknet_access_log_Aug28.gz
gunzip dataset/raw/ClarkNet-HTTP/clarknet_access_log_Sep4.gz
```

### Extract

By default, we extract the whole directory and merge it into one **weekday-only** second-level CSV. Weekends are skipped, so two weeks are compressed into `10` weekdays:

```bash
python pipelines/traffic/extract_clarknet_requests.py
```

Default output:

- `dataset/processed/traffic/ClarkNet-HTTP/clarknet_access_log_aug28_sep10_weekdays_requests_per_second.csv`

To include weekends:

```bash
python pipelines/traffic/extract_clarknet_requests.py --include-weekends true
```

### Plot

Pass the dataset name:

```bash
python pipelines/traffic/plot_daily_minute_curves.py --data clarknet
```

This reads from:

- `dataset/processed/traffic/ClarkNet-HTTP`

and writes plots to the same directory:

- `dataset/processed/traffic/ClarkNet-HTTP`

## Build Locust 10s injection schedules

Convert the extracted **second-level** traffic CSV (`timestamp,request_count`) into a **10-second injection schedule** that Locust can use directly.

### Run

Prerequisite: you have completed the extraction steps above and produced the two second-level CSV files:

- `dataset/processed/traffic/ClarkNet-HTTP/clarknet_access_log_aug28_sep10_weekdays_requests_per_second.csv`
- `dataset/processed/traffic/NASA-HTTP/NASA_access_log_Aug95_19950807_19950820_weekdays_requests_per_second.csv`

Generate schedules:

```bash
# By default, generate both clarknet + nasa schedules
python pipelines/traffic/build_users_schedule_10s.py

# Or generate a single dataset
python pipelines/traffic/build_users_schedule_10s.py --data clarknet
python pipelines/traffic/build_users_schedule_10s.py --data nasa
```

Default scaling strategy:

- Align the global peak `10s avg RPS` to `1000 users`
- Scale all other time points proportionally
- The resulting schedule has a peak of `1000 users`

### Output

By default, schedules are written under `schedules/` in each dataset directory:

- `dataset/processed/traffic/ClarkNet-HTTP/schedules/clarknet_users_10s_peak_u1000.csv`
- `dataset/processed/traffic/NASA-HTTP/schedules/nasa_users_10s_peak_u1000.csv`

The `peak` in the default filename indicates it is generated by aligning to the global peak `10s avg RPS`.

The exported `*_users_10s_*.csv` files are dynamic injection inputs for distributed Locust when running Online Boutique.

The corresponding `*_meta_*.json` records:

- `peak_rps`
- `peak_users`
